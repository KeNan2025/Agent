## 一、赛题核心问题分析

### 1.1 问题定义

本赛题要求构建一套**基于大语言模型与 Agentic AI 的上市公司扫雷预警系统**，核心能力包括：

| 维度 | 目标 | 核心难点 |
|------|------|----------|
| **概率预测** | 预测上市公司未来 30/60/90 天内被问询的概率 | 正样本极稀疏（<5%）、时序依赖、多窗口预测 |
| **风险归因** | 识别触发监管问询的关键风险因素并溯源至公告证据 | 长文本理解、跨文档关联、LLM 幻觉控制 |
| **可解释报告** | 生成包含风险标签、诱因、历史案例、证据片段的预警报告 | 证据闭环、专家可复核、解释有效性量化 |
| **Agent 编排** | 构建自主规划、可追踪、可复现的多智能体协作 Pipeline | 动态路由、工具调用一致性、推理链可溯源 |

### 1.2 技术指标要求（逐项分析可行性）

| 指标 | 目标值 | 难度评估 | 达标策略 |
|------|--------|----------|----------|
| 预测 AUC（60天） | ≥ 0.75 | ★★★☆ | 多模型融合 + 语义特征增强；学术文献中类似任务 AUC 在 0.70-0.80 |
| Top 10% 覆盖率 | ≥ 35% | ★★★☆ | 概率校准 + 排序优化（LambdaRank）|
| 二分类 F1 | ≥ 0.65 | ★★★★ | 代价敏感学习 + 最优阈值搜索（不平衡条件下 F1≥0.65 有挑战）|
| 关注点分类准确率 | ≥ 80% | ★★☆☆ | Few-shot + 标签体系与赛题标注对齐 |
| 证据片段召回率 | ≥ 85% | ★★★☆ | RAG 检索 + LLM 精排 + 多轮校验 |
| 案例 Top-5 命中率 | ≥ 70% | ★★☆☆ | 混合检索（语义+规则）+ 多维相似度 |
| 推理链可追踪率 | 100% | ★☆☆☆ | LangGraph 内建 checkpointing + LangSmith |
| 解释有效性 | ≥ 85 分 | ★★★★ | LLM-as-Judge 自评 + 模板化归因结构 |

### 1.3 数据资源分析

| 数据集 | 规模 | 关键特点 | 处理策略 |
|--------|------|----------|----------|
| 监管问询函及回复 | 1,000-3,000 份 | **核心标签来源**；含问询类型、关注点标签 | 构建正样本 + 案例库 + Few-shot 示例 |
| 上市公司公告 | 30,000-50,000 份 | 长文本（年报数百页）；类型多样 | 章节级切分 + 层级摘要 + 结构化表格提取 |
| 结构化财务数据 | 全样本覆盖 | 三大报表 + 估值 + 行业 | 直接入库，计算衍生指标 |
| 标签与评测 | 7:1:2 切分 | 含人工标注的关注点、证据、案例参考答案 | 严格时序切分，训练/验证/测试无泄露 |

**数据规模研判**：以 A 股约 5000 家上市公司 × 5 年 × 季度频率 ≈ 100,000 条样本为上界，问询正样本约 1,000-3,000 条（正样本率 1-3%），属于**中小规模严重不平衡数据集**，这直接影响模型选型（TabPFN-2.5 的最优区间）。

## 三、四大任务详细技术方案

### 任务一：监管问询语料解析与风险关注点抽取

#### 3.1.1 核心挑战与解题思路

```
挑战：长文本（年报数百页）中精准抽取监管关注点，避免幻觉和错误归因
思路：不做端到端的"文档→标签"，而是分三步走——
      ① 结构化解析（确定性）→ ② RAG 检索定位（半确定性）→ ③ LLM 抽取（可控生成）
```

#### 3.1.3 监管关注点标签体系（基于真实监管实践）

基于中国证券交易所问询函历史分析，构建三级标签体系：

| 一级标签 | 二级标签 | 典型触发条件（供 LLM Few-shot 使用） |
|----------|---------|--------------------------------------|
| **财务异常** | 收入确认异常 | 收入增速远超行业或应收增速远超收入增速 |
| | 利润质量存疑 | 扣非净利润持续为负但营收增长、经营现金流与净利润严重背离 |
| | 大额资产减值 | 商誉/净资产>30%、应收坏账计提比例偏低 |
| | 毛利率异常 | 毛利率显著偏离行业均值（>2σ），或剧烈波动 |
| **关联交易** | 定价公允性 | 关联交易价格偏离市场价、缺少可比交易参照 |
| | 资金往来 | 与关联方存在大额非经营性资金往来 |
| | 隐性关联 | 未披露的实际控制人关系、一致行动人 |
| **资金问题** | 资金占用 | 大股东/关联方占用上市公司资金 |
| | 违规担保 | 未经审批的对外担保、为关联方提供担保 |
| | 大额预付 | 预付账款异常增长且集中于少数供应商 |
| **经营合理性** | 业绩预告偏差 | 实际业绩与预告偏差超过 30% |
| | 经营突变 | 主营业务突然变更或新增占比极高的业务 |
| | 持续经营风险 | 连续亏损、营运资金为负、审计保留意见 |
| **信息披露** | 前后矛盾 | 不同公告间数据/表述不一致 |
| | 披露不充分 | 重大事项未及时披露或遗漏关键信息 |
| **公司治理** | 高管异动 | 短期内多名董监高辞职、财务总监/审计委员更替 |
| | 控制权争夺 | 股东间公开分歧、举牌/反收购 |
| | 内控缺陷 | 审计师出具否定意见内控报告 |
| **并购重组** | 标的高估值 | 评估增值率异常高、缺乏可比案例 |
| | 业绩承诺风险 | 承诺期临近且完成率持续下滑 |
| **会计争议** | 政策变更 | 会计估计变更导致利润大幅波动 |
| | 收入确认方法 | 完工百分比法争议、总额/净额法选择 |

#### 3.1.4 LLM 抽取方案（关键改进：RAG + Few-shot + 结构化输出）

**改进点**：初版采用简单 Prompt + 自由输出。修订版引入三重保障——

```python
# 改进1：先检索再抽取（RAG），而非直接对全文抽取
def extract_risk_factors(company_code, report_period):
    # Step 1: 先用监管关注点关键词检索相关 Chunk
    query_templates = [
        f"{company_code} 收入确认 异常",
        f"{company_code} 关联交易 定价",
        f"{company_code} 商誉减值 风险",
        # ... 覆盖每个二级标签的关键词模板
    ]
    relevant_chunks = hybrid_search(query_templates, top_k=20)
    
    # Step 2: 对每个检索到的 Chunk 做精准抽取
    for chunk in relevant_chunks:
        result = llm.structured_extract(
            chunk=chunk,
            schema=RiskFactorSchema,  # 结构化输出约束
            few_shot_examples=ANNOTATED_EXAMPLES,  # 赛题标注数据
        )
    return results

# 改进2：使用 LLM 原生结构化输出（tool calling），而非自由文本解析
class RiskFactorSchema(BaseModel):
    """单条风险要素的结构化定义"""
    category: Literal["财务异常","关联交易","资金问题","经营合理性",
                       "信息披露","公司治理","并购重组","会计争议"]
    subcategory: str
    description: str = Field(max_length=100)
    evidence_quote: str  # 必须为原文直接引用
    evidence_source: str  # 章节+页码
    severity: Literal["高", "中", "低"]
    confidence: float = Field(ge=0.0, le=1.0)

# 改进3：Few-shot 示例来自赛题标注数据
ANNOTATED_EXAMPLES = [
    {
        "input": "公司本期营业收入增长45%，但经营活动产生的现金流量净额...",
        "output": RiskFactorSchema(
            category="财务异常",
            subcategory="利润质量存疑",
            description="收入高增长但经营现金流反降，收入质量存疑",
            evidence_quote="本期营业收入较上年同期增长45.2%...",
            evidence_source="2023年年报 第四节 经营情况讨论与分析 P28",
            severity="高",
            confidence=0.88
        )
    },
    # ... 从赛题标注数据中选取 3-5 个高质量示例
]
```

#### 3.1.5 防幻觉机制（增强版）

| 层级 | 策略 | 实现细节 | 初版改进点 |
|------|------|----------|-----------|
| **生成前** | RAG 检索锚定 | 先检索再抽取，LLM 输入只包含相关 Chunk | **新增**：初版无 RAG |
| **生成中** | 结构化输出约束 | LLM tool calling 强制输出 JSON Schema | **改进**：初版为自由文本 |
| **生成中** | Few-shot 对齐 | 使用赛题标注数据作为 Few-shot 示例 | **新增**：初版无 Few-shot |
| **生成后** | 证据回溯校验 | 后处理检查 evidence_quote 是否存在于原文（模糊匹配 + 编辑距离） | 保留并增强 |
| **生成后** | 数值交叉核验 | 抽取的财务数值与结构化数据库比对 | 保留 |
| **生成后** | 双通道交叉验证 | 同一文档用 Qwen3 和 DeepSeek-V3 分别抽取，取交集 | **改进**：初版为同模型不同 Prompt |
| **全局** | Verifier Agent | 独立 Agent 审查抽取结果的逻辑一致性和证据充分性 | **新增** |

### 任务二：上市公司被问询概率预测模型

#### 3.2.1 问题形式化

```

数据特征：
  - 样本量级：~100,000 条公司-季度观测（5000 公司 × 5 年 × 4 季度）
```

**模型选型修订说明**：

| 修订 | 理由 |
|------|------|
| **CatBoost 替代 XGBoost** | CatBoost 对类别特征（行业、审计意见类型、公告类型）内建支持，无需手动编码；在不平衡数据上表现更稳定；且在 2025 年多项 Kaggle 竞赛中表现优于 XGBoost |
| **TabPFN-2.5 替代 TabNet** | TabPFN-2.5 是 2025 年 Nature 发表的表格基础模型，在 ≤10 万样本数据集上达到 SOTA，100% 胜率超越默认 XGBoost；零调参即可使用；本赛题样本量（~10万）在其最优区间内 |
| **LightGBM 保留** | 训练快，高维稀疏特征处理好，与 CatBoost 形成互补 |

#### 3.2.4 样本构建与训练策略（修订版）

**样本粒度选择**：

```
推荐方案：季度级快照（每季度末对每家公司生成一条观测）
理由：
  - 年报/季报发布节奏为季度级
  - 季度级可平衡样本量与特征时效性
  - ~5000 公司 × 20 季度（5年）= ~100,000 条观测
```

**时序切分（严格禁止未来泄露）**：

```
Year 1-3 ──────────── Year 4 ────── Year 5
│                      │              │
│   训练集 (60%)       │ 验证 (15%)   │ 测试集 (25%)
│                      │              │
└──── 特征计算截止 ────┘              │
      ↑ 所有特征只用到此时刻之前的数据   │
                                      └── 最终评估
```

**类别不平衡处理（修订版，去除 SMOTE）**：

| 策略 | 初版 | **修订版** | 修订理由 |
|------|------|-----------|----------|
| 过采样 | SMOTE | **去除** | SMOTE 在时序数据中会生成时间上不合理的伪样本，且可能夸大特征分布 |
| 欠采样 | 分层欠采样 | **时序感知分层欠采样** | 按行业+时间窗口分层，保证每个时间段都有足够正负样本 |
| 代价函数 | Focal Loss | **非对称代价矩阵（10:1）** | 将"漏检"（FN）代价设为"误报"（FP）的 10 倍；Focal Loss 对 GBDT 无效 |
| 类权重 | 类别权重 | **scale_pos_weight = neg/pos** | CatBoost/LightGBM 原生支持 |
| 阈值 | F1 最优 | **F1 + 召回率联合优化** | 赛题同时考核 F1 和 Top 10% 覆盖率 |
| 评估指标 | AUC-ROC | **AUC-ROC + AUC-PR + F1** | AUC-PR 在严重不平衡下比 ROC 更有辨别力 |

#### 3.2.5 模型可解释性（增强版）

| 层级 | 方法 | 用途 |
|------|------|------|
| **全局** | SHAP Summary Plot | 展示 Top-20 重要特征，理解模型整体逻辑 |
| **局部** | SHAP Waterfall | 单公司概率归因，找到最关键的 3-5 个驱动因素 |
| **规则** | 从 CatBoost 提取高置信决策路径 | 可复核的风险规则（如"应收增速 > 收入增速 × 2 且 商誉占比 > 30% → 高风险"）|

| **LLM归因** | 将 SHAP Top 特征 + 证据传给 LLM 生成自然语言归因 | 最终报告中的文字说明 |

### 任务三：自主规划型扫雷 Agent 与 Skill 编排

#### 3.3.1 设计原则（修订版核心改进）

初版问题：Agent 的"自主规划"描述流于形式，实际是固定的 DAG 流程。
修订方案：引入 **ReAct 动态规划** + **条件路由**，实现真正的自主决策。

```
核心区别：

初版（伪自主规划）：
  公告研读 → 财务检测 → 概率预测 → 案例检索 → 归因解释
  （固定顺序，所有公司走同一路径）

修订版（真自主规划）：
  Planner 先做"初筛假设" → 根据假设选择性调用子 Agent → 中间结果反馈
  → Planner 动态调整后续分析路径 → 适应性归因
  （不同类型风险走不同分析路径）
```

#### 3.3.2 LangGraph StateGraph 设计

```python
from langgraph.graph import StateGraph, END
from langgraph.checkpoint.memory import MemorySaver

class ScanState(TypedDict):
    company_code: str
    window_days: int
    # 动态规划状态
    risk_hypothesis: list[str]       # Planner 生成的风险假设
    analysis_plan: list[str]         # 当前分析计划（可动态修改）
    completed_steps: list[str]       # 已完成步骤
    # 各 Agent 产出
    announcement_analysis: dict | None
    financial_anomalies: dict | None
    graph_risks: dict | None
    similar_cases: list[dict] | None
    prediction_result: dict | None
    # 最终输出
    report: str | None
    trace_log: list[dict]            # 推理链路日志

# 构建状态图
graph = StateGraph(ScanState)

# 节点定义
graph.add_node("planner", planner_node)          # 初始规划
graph.add_node("announcement_agent", ann_node)    # 公告研读
graph.add_node("financial_agent", fin_node)       # 财务异常
graph.add_node("graph_agent", graph_node)         # 图谱分析
graph.add_node("predictor", predict_node)         # 概率预测
graph.add_node("case_agent", case_node)           # 案例检索
graph.add_node("replan", replan_node)             # 动态重规划
graph.add_node("attribution_agent", attr_node)    # 归因报告

# 关键：动态路由
graph.add_conditional_edges(
    "planner",
    route_by_hypothesis,  # 根据风险假设动态选择下一步
    {
        "financial_focus": "financial_agent",    # 财务异常类→优先财务分析
        "disclosure_focus": "announcement_agent",# 信披类→优先公告研读
        "governance_focus": "graph_agent",       # 治理类→优先图谱分析
        "parallel_all": "parallel_fork",         # 不确定→并行所有
    }
)

# 中间重规划节点
graph.add_conditional_edges(
    "replan",
    should_continue_or_finalize,
    {
        "need_more_analysis": "planner",  # 发现新线索→回到 Planner 重规划
        "ready_to_predict": "predictor",  # 信息充足→进入预测
    }
)
```

#### 3.4.2 评估体系（新增）

初版缺少系统性评估方案。赛题要求"逻辑解释有效性由专家评委或 LLM 辅助评审打分"，需要设计：

**LLM-as-Judge 评估框架**：

```python
JUDGE_PROMPT = """
你是一名资深金融监管审查专家。请评估以下预警报告的质量。

【预警报告】
{report_content}

【评分维度】（每项 0-100 分）

1. 风险识别准确性（25%）：风险因素是否与真实问询原因一致？
2. 证据充分性（25%）：每条风险是否有原文证据支撑？证据是否真实可溯源？
3. 逻辑链完整性（25%）：从数据异常到风险判断的推理链是否完整、无跳跃？
4. 案例匹配合理性（15%）：相似案例是否确实在关键维度上相似？
5. 业务实用性（10%）：投研/风控人员能否据此报告采取具体行动？

请输出 JSON：
{
  "scores": {"accuracy": X, "evidence": X, "logic": X, "cases": X, "utility": X},
  "weighted_total": X,
  "issues": ["具体问题1", "问题2"],
  "suggestions": ["改进建议1", "建议2"]
}
"""
```

**消融实验设计**：

| 实验 | 去除/替换内容 | 预期影响 | 目的 |
|------|-------------|----------|------|
| Ablation-1 | 去除 LLM 语义特征 | AUC 下降 3-5% | 量化语义特征贡献 |
| Ablation-2 | 去除知识图谱特征 | AUC 下降 1-3% | 量化关联风险贡献 |
| Ablation-3 | 去除历史监管特征 | AUC 下降 2-4% | 量化历史信号贡献 |
| Ablation-4 | 单模型 vs 集成 | F1 下降 3-8% | 验证融合必要性 |
| Ablation-5 | 固定Pipeline vs 动态规划 | 解释有效性降 5-10分 | 验证自主规划价值 |
| Ablation-6 | 去除防幻觉机制 | 证据召回率降 10-15% | 验证校验机制必要性 |

**基线对比方案**：

| 基线 | 方法 | 对比意义 |
|------|------|----------|
| Baseline-1 | 纯规则模型（财务阈值规则） | 与传统方法对比 |
| Baseline-2 | 纯 GBDT（仅结构化特征） | 量化 LLM 增量价值 |
| Baseline-3 | 纯 LLM 直接预测（Zero-shot） | 量化结构化建模价值 |

## 四、关键技术创新点（修订版）

### 4.1 "监管思维链"（Regulatory Chain-of-Thought）

```
传统方法：   指标超阈值 → 机械预警（高误报率）
初版方案：   LLM 模拟监管逻辑 → 预测（方向正确但笼统）
修订方案：   分析赛题问询函数据 → 提炼"监管关注模式"
            → Few-shot 注入 → LLM 按模式推理 → 预测+归因
```

创新点：不是泛泛让 LLM "扮演监管人员"，而是从历史问询函中**数据驱动地提炼监管关注模式**，作为 LLM 推理的结构化先验。

### 4.2 知识图谱增强的风险传导建模（新增）

```
传统方法：   孤立分析每家公司
修订方案：   
  公司A（已被问询）──关联交易──→ 公司B（风险传导）
       │                              ↑
       └──同一实控人──→ 公司C ────供应链──┘
       
  图谱特征：节点度、PageRank、关联圈内被问询比例
  → 捕获"一家出事，关联方被问询"的监管模式
```

### 4.3 TabPFN-2.5 + GBDT 混合集成（新增）

```
传统方案：   GBDT 三模型 Stacking（同质化较高）
修订方案：   TabPFN-2.5（基础模型范式）+ CatBoost/LightGBM（GBDT范式）
            → 异质集成，互补性更强
            → TabPFN 在小样本上的优势弥补正样本稀缺的劣势
```

### 4.4 动态规划 Agent（真自主决策）

```
初版：固定 DAG（所有公司走同一路径）
修订：Planner 基于初筛假设动态路由 → 中间 Replan 节点反馈调整
     → 不同风险类型走不同分析路径
     → LangGraph 条件边实现，保证可追踪可复现
```

### 4.5 全链路三级穿透归因

```
概率 78.3%
  └→ SHAP 分解：收入异常 +0.15（最大贡献）
       └→ 语义特征：risk_factor_count["财务异常"] = 3
            └→ LLM 抽取结果：
                 └→ 原文证据："本期营业收入较上年同期增长45.2%..."
                      └→ 来源：2024年三季报 P12 第4段
```
