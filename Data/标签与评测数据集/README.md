# 标签与评测数据集使用指南 (Labels & Evaluation)

本目录（`标签与评测数据集/`）包含用于算法模型训练、验证和评测的核心靶标数据与官方参考答案。

## 1. 包含文件清单

| 文件名 | 说明 | 用途 |
|---|---|---|
| `dataset_split_labels.csv` | 包含全部 1957 家样本公司的基本标签（1=负面/有问询，0=正面/无问询），并增加了 `split` 字段，严格按照 7:1:2 的比例切分为 Train（训练集）、Validation（验证集）、Test（测试集）。 | 用于区分不同生命周期的样本集，防止数据穿越。选手应仅使用 Train 集合的样本进行算法调优与 Agent 提示词编写。 |
| `evaluation_ground_truth.csv` | 包含在 Validation 和 Test 集中所有存在真实问询记录公司的“机器辅助官方参考答案（Pseudo-Labels）”。包含公司代码、问询日期及问询关注点提取的 JSON。 | 用于评估选手 Agent 对复杂长文本的“语义抽取”、“归因解释”和“逻辑匹配”能力的客观定量评测。 |

## 2. 为什么是最佳实践？

*   **7:1:2 的严谨切分**：我们保证了同一家公司的标签不会横跨不同的切片。验证集和测试集加起来包含将近 600 家公司的数据，足以检验任何模型泛化能力。
*   **如何弥补“人工标注成本高”的问题**：要求专家去研读上千篇问询函并抽取重点，不仅成本极高且容易引入主观误差。本数据集巧妙利用了前期跑数据时存留的 **高规格大语言模型 (Teacher Model) 的结构化提取结果** 作为 Ground Truth。文件里提供的 `regulatory_focus_points_json` 精准总结了交易所发函质询的具体条目。

## 3. 对参赛选手的要求与使用建议

1.  **概率预测评估 (AUC / F1)**：
    使用 `dataset_split_labels.csv` 作为标准靶标。选手的输出概率分数必须与此表对齐，使用标准的 `sklearn.metrics.roc_auc_score` 跑出 AUC，评估扫雷的准度。
2.  **归因与语义提取评估 (ROUGE / 语义相似度)**：
    选手提交模型对测试集公司出具的“预警/风险扫雷报告”时，赛事评委可以利用 `evaluation_ground_truth.csv` 里的 JSON 关注点，结合 Embedding 相似度或者 ROUGE-L 指标，去自动化给选手模型的“解释合理性”打分！
    （例如，如果官方答案提取了“应收账款周转率异常”，选手的 Agent 也成功把“应收账款”提炼为主要诱因，即可获得该维度的满分。）
