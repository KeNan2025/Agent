## 1. 精准定位：经济学C刊要什么？

### 1.1 经济学C刊与计算机论文的根本区别

| 维度 | 计算机/工程论文 | 经济学C刊论文 |
|------|---------------|--------------|
| **核心问题** | 模型精度有多高？ | 经济机制是什么？政策含义是什么？ |
| **创新点来源** | 算法改进、架构创新 | 新发现的事实、因果关系识别、理论解释 |
| **模型角色** | 核心贡献 | 分析工具（方法服务于问题） |
| **评估标准** | ACC、AUC、F1 | 因果识别可信度、经济显著性、稳健性 |
| **最重要章节** | Methodology | 理论分析 + 实证设计 |
| **论文篇幅** | 8-12页 | 15-25页（2万字左右） |
| **审稿人背景** | 算法专家 | 金融/会计/计量经济学家 |

### 1.2 经济学C刊审稿人最关心的五个问题

```
Q1: 你这个问题的经济学意义何在？为什么重要？
    → 需要在引言中用经济学逻辑回答，而非技术叙事

Q2: 你的核心发现是什么？是不是常识/已知事实？
    → 需要清晰的、反直觉或具有政策含义的核心结论

Q3: 内生性问题处理了吗？识别策略是否可信？
    → 这是经济学论文与工科论文最大的分水岭
    
Q4: 你的数据和方法有多可靠？
    → 需要详细的描述性统计、相关性分析、稳健性检验

Q5: 这篇文章告诉我们应该做什么？
    → 明确的政策建议和理论贡献
```

### 1.3 你的独特优势

作为经济学背景的研究者，你有计算机背景者不具备的"比较优势"：

- **懂制度**：你能讲清楚中国监管问询的制度背景（交易所问询函制度、注册制改革等）
- **懂机制**：你能从信息经济学、委托代理理论解释"为什么会被问询"
- **懂政策**：你的论文能提出监管部门、投资者、上市公司三方都有意义的政策建议
- **懂因果**：你会在意"相关性 ≠ 因果性"，主动处理内生性问题

> **核心竞争力**：你不是在"调参数"，而是在"研究一个重要的经济学问题"。

## 2. 选题破题：你的论文要回答什么问题

### 2.1 核心研究问题（建议三选一聚焦）

围绕赛题，以下是三个可以支撑C刊论文的研究问题（按推荐排序）：

#### 方案一：⭐⭐⭐⭐⭐（强推荐）

**"上市公司被监管问询的经济前因——基于多源信息的机器学习预测与可解释性分析"**

- **核心问题**：究竟是什么经济因素驱动了公司被监管问询？传统单指标模型的可预测性有多强？引入文本语义信息后，预测能力提升多少？
- **角度**：**前因识别 + 增量贡献**（"信息含量"视角）
- **经济学意义**：检验市场信息是否已经充分反映在公开财务数据中，评估大语言模型作为"软信息"提取手段的经济价值
- **适合你**：偏向金融经济学、实证资产定价方向

#### 方案二：⭐⭐⭐⭐

**"监管问询预测中的可解释性与市场学习机制"**

- **核心问题**：可解释的风险归因能否帮助市场参与者更好理解监管关注点？与纯概率输出相比，可解释报告的增量信息价值有多大？
- **角度**：**信息传递 + 市场效率**视角
- **经济学意义**：监管问询的信息含量和传递机制研究
- **适合你**：偏向行为金融、市场微观结构方向

#### 方案三：⭐⭐⭐⭐

**"大语言模型能否改善上市公司风险预测？——兼论传统财务指标与文本语义信息的信息含量"**

- **核心问题**：LLM提取的文本语义特征相对于传统财务指标，是否具有**增量预测能力**？
- **角度**：**信息含量比较**视角（类似会计学"价值相关性"文献）
- **经济学意义**：对"软信息"与"硬信息"在风险预测中的相对重要性做量化比较
- **适合你**：偏向会计学、信息披露方向

> **重点推荐方案一**。原因是：它最接近经济学C刊的核心关切——**识别重要经济变量之间的关系并量化其经济显著性**。

## 3. 理论框架：把技术问题翻译成经济学语言

这是经济学论文的**灵魂**。你不能只写"我用机器学习做了个预测模型"，而要从经济学理论出发提出假说。

### 3.1 制度背景分析（Section 2 的核心内容）

建议花至少1页篇幅系统介绍中国A股监管问询制度，这是经济学论文的"必修课"：

```
要点1：中国问询函制度的演进
  - 2014年交易所"分类监管"改革 → 2018年全面推行问询函制度
  - 注册制改革背景下，问询函成为事前审核向事中事后监管转变的核心工具
  
要点2：问询函的分类与触发机制
  - 年报问询函（最常见，针对年报信息披露问题）
  - 关注函（针对重大事项、媒体报道等）
  - 重组问询函（并购重组审核）
  - 问询函的法律性质（非行政处罚，但具有市场信号效应）

要点3：问询函的信息经济学含义
  - 问询函是监管者向市场传递的"软警告"信号
  - 被问询公司的股价通常在公告日附近产生显著负向超额收益
  - 问询函回复后，公司信息披露质量通常会改善
  
要点4：中国市场的特殊性
  - 中国A股市场以散户为主，信息不对称更严重
  - 问询函的监管力度和频率与市场环境密切相关
  - 不同行业、不同市场板块的问询概率存在系统差异
```

### 3.2 理论基础（构建你的分析框架）

建议从以下理论中选取 2-3 个作为论文的理论支撑：

#### 理论一：信息不对称与信号传递理论（Akerlof, 1970; Spence, 1973）

```
核心论点：
  上市公司与投资者之间存在严重的信息不对称。监管问询是监管者
  发现"柠檬问题"后向市场发出的纠偏信号。
  
在本论文中的应用：
  公司特征（财务指标、公告文本等）反映了内部人掌握的信息；
  当这些信息异常时，监管者有更高概率发出问询。
  因此，公司特征可以"预测"问询概率。
  
可检验的假说：
  H1a：财务指标异常程度越大，公司被问询的概率越高。
  H1b：公告文本中隐藏的"软信息"（语气、可读性等）比硬信息
       具有增量预测能力。
```

#### 理论二：信息披露质量与公司治理（Healy & Palepu, 2001）

```
核心论点：
  高质量的财务信息披露降低信息不对称和代理成本。
  信息披露质量低是触发监管问询的重要原因。
  
在本论文中的应用：
  信息披露质量可以通过多个维度衡量：财务重述历史、可读性、
  及时性、MD&A信息含量等。这些变量应影响问询概率。
  
可检验的假说：
  H2：信息披露质量越差的公司，被问询概率越高。
```

#### 理论三："软信息"的增量价值理论（Liberti & Petersen, 2019, JEL）

```
核心论点：
  "硬信息"（可量化的财务指标）和"软信息"（文本、语调、语义）
  在风险评估中具有互补性。软信息在处理非结构化、情境依赖的
  问题上尤其有价值。
  
在本论文中的应用：
  大语言模型可以高效提取公告文本中的"软信息"。
  这些软信息特征应能显著提升预测精度，尤其是对于"边界案例"。
  
可检验的假说：
  H3：软信息特征对监管问询预测具有显著的增量贡献，
       且该贡献在信息不对称更严重的公司中更强。
```

#### 理论四：监管经济学（Becker-Stigler Model）

```
核心论点：
  监管者的资源是有限的，会优先选择"信号最强"的公司进行检查。
  
在本论文中的应用：
  监管者的问询行为不是随机的，而是有系统性选择偏好的。
  理解这种选择偏好，是预测的前提。
  
可检验的假说：
  H4：监管问询行为具有行业集群性（同行业示范效应）和历史持续性。
```

### 3.4 可检验的研究假说

将上面的理论转化为至少 3-5 个可检验的假说：

```
假说1（硬信息假说）：传统财务指标（Beneish M-Score、应收增速异常等）
    与监管问询概率显著正相关。该效应在控制行业和时间固定效应后依然成立。

假说2（软信息假说）：大语言模型提取的文本语义风险特征在控制财务指标后，
    对监管问询概率具有显著的增量预测能力。

假说3（异质性假说）：软信息的增量预测能力在以下公司中更强：
    (a) 小市值公司（信息不对称更严重）；
    (b) 高成长行业（财务指标的噪声更大）；
    (c) 机构投资者持股比例低的公司（外部监督较弱）。

假说4（时间维度假说）：监管问询具有正的时间序列自相关性，
    即历史被问询的公司更有可能再次被问询。

假说5（行业扩散假说）：同一行业内近期被问询的公司数量越多，
    其他公司被问询的概率也越高（监管关注度的行业溢出效应）。
```

## 4. 研究设计：实证策略是论文的脊梁

这是经济学论文中**审稿人看得最仔细的部分**。你需要清晰说明：

### 4.1 样本构建策略

#### 时间窗口设计

```
关键选择：使用多个时间窗口（30/60/90天）还是一个？

推荐方案：以60天为主窗口，30天和90天做稳健性检验。
  - 60天：约等于2个月，是"事件发生前"的合理窗口
  - 30天：检验短期预测能力
  - 90天：检验预测能力的衰减速度

时间切分（严格防止未来信息泄露）：
  ┌─────────┬─────────┬─────────┬─────────┐
  │  训练集  │ 验证集   │  测试集  │         │
  │  Year1-3 │ Year4   │  Year5  │  时间→  │
  │ (60%)   │ (15%)   │ (25%)   │         │
  └─────────┴─────────┴─────────┴─────────┘
  注意：特征计算只用到截止时点之前的信息！
```

#### 样本粒度

```
推荐：公司-季度面板（company-quarter panel）
  - 每季度末对所有上市公司生成一条观测
  - 特征使用截至该季度的最新可得数据
  - 标签：该季度后60天内是否被问询
  - 约 5000 公司 × 20 季度 ≈ 100,000 条观测
  
可选：公司-月度面板（更细粒度，但财务数据更新频率为季度）
  - 约 5000 公司 × 60 月 ≈ 300,000 条观测（大部分特征重复）
```

### 4.2 模型设定（实证框架）

#### 基准模型：Logistic回归（经济学论文的标准起点）

```
P(Inquiry_{i,t+Δt}=1 | X_{i,t}) = Λ(α + β₁Financial_{i,t} + β₂Text_{i,t} 
                                       + β₃Market_{i,t} + β₄Governance_{i,t}
                                       + γ_t + δ_j + ε_{i,t})

其中：
  - Λ(·) 为 Logistic 累积分布函数
  - Financial 为财务特征向量
  - Text 为文本语义特征向量
  - Market 为市场特征向量
  - Governance 为公司治理特征向量
  - γ_t 为时间固定效应（控制宏观经济和监管政策变化）
  - δ_j 为行业固定效应（控制行业监管差异）
  - ε_{i,t} 为公司层面聚类稳健标准误
```

#### 为什么Logistic是必选项？

```
在经济学C刊中，Logistic回归是"默认语言"：
  ① 经济学审稿人不需要额外解释就知道怎么读系数
  ② 可以报告边际效应，具有经济学意义
  ③ 是后续所有模型的自然基准

而如果一上来就是XGBoost、神经网络，审稿人会觉得：
  "这是在炫技而不是在搞经济学研究"

#### 关键计量问题处理

| 计量问题 | 你怎么处理 | 怎么写进论文 |
|---------|-----------|-------------|

## 6. 模型方案：从计量经济学视角驾驭机器学习

### 6.1 分层建模策略（经济学论文推荐方案）

不要一上来就"跑机器学习"，而是做一个**"线性到非线性"的金字塔结构**：

```
第四层：XGBoost / LightGBM（最优预测性能）
    ↑
第三层：Lasso / Ridge / Elastic Net（变量选择 + 正则化）
    ↑
第二层：Probit / Logit（处理非线性 + 边际效应报告）
    ↑
第一层：Linear Probability Model（LPM）（最简单的基准）
    ↑
起点：基础描述性统计 + 相关系数矩阵
```

**论文中实际呈现方式**：重点写第二层（Logit）和第四层（ML），第一层和第三层作为稳健性检验放在附录。

### 6.2 如何向经济学审稿人解释机器学习

这是关键——你不能假设审稿人懂XGBoost，但也不能过度简化到失去准确性。

```
建议在论文中增加的"方法说明"段落（约半页）：

"在基准Logistic回归之外，本文进一步采用集成学习方法（Gradient 
Boosting Decision Tree, GBDT）评估预测性能的上限。GBDT通过序贯
迭代拟合前一轮残差，可自动捕捉特征间的高阶交互效应和非线性关系。
相较于传统线性模型，GBDT具有以下优势：（1）无需预设特征交互形式；
（2）对缺失值和异常值有较强鲁棒性；（3）内建特征重要性排序机制。
本文选择LightGBM作为GBDT的实现框架，采用早停（early stopping）

### 6.3 预测模型的技术选型

| 模型 | 在论文中的角色 | 优点 |
|------|-------------|------|

### 6.4 样本不平衡处理（经济学视角）

```
经济学审稿人的关注点：
  "你的正样本只有2%，模型是不是在学'预测多数类'？
  为什么不用更自然的抽样方法？"

论文中建议的做法：
  ① 不做SMOTE：SMOTE生成人工样本，在时序数据中会产生
     逻辑上不合理的"未来样本"，经济学期刊审稿人不接受
     
  ② 使用"加权Logistic回归"：
     - 权重 = 负样本数/正样本数

     - 不用默认的0.5，而是用Youden Index选择最优阈值

     - 按行业和时间分层，保证训练集中的正样本占比合理

### 6.5 消融实验设计（经济学论文版）

在经济学论文中，不叫"Ablation Study"，而是用以下框架：

  → 分别用30天、60天、90天窗口

```

### 9.4 理论机制（需要你重点发挥的部分）

以下是你可以使用的理论机制（选2-3个）：

```
机制1：信号博弈
  - 公司选择信息披露的"真实度"和"充分度"
  - 监管者观察到信号（公告、财务数据）后判断是否问询
  - 信号越"异常"（偏离行业均值、历史趋势），越可能被问询
  
机制2：信息披露的"可验证性"差异
  - 硬信息（财务数据）容易验证，造假成本高
  - 软信息（文字描述）难以验证，是"隐藏坏消息"的渠道
  - 监管者会特别关注那些"软信息与硬信息矛盾"的公司
  
机制3：监管资源分配的"优先原则"
  - 监管者资源有限，会优先关注"信号最强"的公司
  - 信息不对称严重的公司（小公司、少分析师覆盖）是重点
```

### 10.1 审稿意见预测

| 审稿意见类型 | 典型表述 | 你的应对策略 |
|-------------|---------|-------------|

| **经济意义质疑** | "AUC提升0.06在经济上显著吗？" | 报告边际效应和相对概率变化，展示经济显著性 |

| **数据质疑** | "训练集和测试集是否真正独立？" | 详细说明时间窗口划分，展示严格防泄漏措施 |
| **软信息贡献质疑** | "文本特征和财务特征高度相关，是否能真正分离？" | 做VIF检验、相关性分析、LASSO变量选择的稳定性检验 |
| **应用场景质疑** | "预测结果在实践中如何应用？" | 详细说明使用场景（投资风控、合规巡检、投研辅助） |
| **新意不足** | "这和已有风险预警模型有何本质区别？" | 强调"软信息的增量价值"和"可解释性归因"两个创新点 |

## 11. 论文写作时间规划

按"逐步推进、预留修改时间"的原则，建议3个月完成：

```
第1-2周：文献工作
  - 系统阅读30-40篇核心文献，下载并分类
  - 提炼文献评述的关键论点
  - 确定论文的定位和理论框架

第3-4周：数据处理
  - 赛题数据清洗和结构化
  - 构建公司-季度面板
  - 计算财务指标（Beneish M-Score、Altman Z-Score等）
  - LLM文本特征提取与质量校验

第5-6周：模型实现
  - 基准Logistic回归（验证基础假说）
  - LightGBM预测模型（验证预测性能）
  - SHAP可解释性分析
  - 编写所有表格和图标代码

第7周：初稿撰写
  - 从"实证结果"部分开始写（最容易的部分）
  → 引言 → 理论分析 → 文献综述 → 数据与变量 → 研究设计 → 结论
  - 每天写1-2节，一周完成全文初稿

第8周：自我修改
  - 检查逻辑一致性
  - 补充缺失的稳健性检验
  - 反复修改引言（最重要的部分）

第9周：同行反馈
  - 请导师/同学读稿，收集反馈
  - 针对审稿意见修改
  - 凝练摘要和结论

第10周：最终润色
  - 格式检查（图表编号、参考文献格式）
  - 语言润色
  - 提交前最后检查
```
